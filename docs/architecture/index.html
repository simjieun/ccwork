시나리오 A — 노트 생성

"+ 새 노트" 클릭 → 작성 → 저장까지의 전체 흐름

sequenceDiagram actor U as 사용자 participant App participant NE as NoteEditor participant CTX as NotesContext participant API as api/notes.ts participant DB as JSON Server U->>App: "+ 새 노트" 클릭 App->>NE: isCreating=true, selectedNoteId=null U->>NE: 제목·내용 입력 U->>NE: "저장" 클릭 NE->>CTX: createNote(title, content) CTX->>API: createNote({title, content}) API->>DB: POST /notes DB-->>API: 201 Created + Note API-->>CTX: newNote CTX->>CTX: setNotes([...prev, newNote]) CTX-->>NE: notes 업데이트 NE->>App: onDone() App->>App: isCreating=false

시나리오 B — 노트 수정

노트 선택 → 내용 변경 → 저장

sequenceDiagram actor U as 사용자 participant NI as NoteItem participant App participant NE as NoteEditor participant CTX as NotesContext participant API as api/notes.ts participant DB as JSON Server U->>NI: 노트 클릭 NI->>App: onSelect(id) App->>NE: selectedNoteId=id, isCreating=false NE->>CTX: notes.find(n => n.id === id) CTX-->>NE: selectedNote NE->>NE: setTitle / setContent U->>NE: 내용 수정 후 "저장" NE->>CTX: updateNote(id, {title, content}) CTX->>API: updateNote(id, {...}) API->>DB: PATCH /notes/:id DB-->>API: 200 OK + updated Note API-->>CTX: updatedNote CTX->>CTX: setNotes(prev.map(...)) NE->>App: onDone()

시나리오 C — 노트 삭제

삭제 버튼 클릭 → Context → API → 상태 갱신

sequenceDiagram actor U as 사용자 participant NI as NoteItem participant CTX as NotesContext participant API as api/notes.ts participant DB as JSON Server U->>NI: "삭제" 버튼 클릭 NI->>CTX: deleteNote(id) CTX->>API: deleteNote(id) API->>DB: DELETE /notes/:id DB-->>API: 200 OK API-->>CTX: void CTX->>CTX: setNotes(prev.filter(n => n.id !== id))
